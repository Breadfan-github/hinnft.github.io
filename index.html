LIME

Features NFT ownership based access to exclusive content, custom ERC721 & ERC20 tokens, decentralized Swap with WETH converter, NFT minting dAPP, NFT marketplace, faucet, & content uploading via IPFS.

  • View Live site (Goerli)
  • Github
  • Stack: Solidity, Javascript ES6, Hardhat, Ethers.js, IPFS, React.js(Hooks) + React bootstrap

My first real world unguided project.

This project is inspired by the OnlyFans & Patreon modal, to give users exclusive access to content based on their LIME NFT(custom ERC721 token) ownership. Users can receive $LIME (custom ERC20 token) from the dAPP 'Faucet' and choose to mint a NFT on the 'Launchpad' or purchase one from the 'Marketplace'. The user can also obtain additional $LIME from the 'Swap' for GoerliETH. Users can also list & trade their LIME NFTs on the 'Marketplace' from the 'My Owned' tab. Users can now access content and tip on the 'Home' page after obtaining (1) LIME NFT.

For now, the 'Upload Content' page/function is only limited to the Admin. A multi-creator system upgrade can be implemented later on.

I decided to go with the Hardhat & Ethers.js route for this project due to the clean syntax and large community support. This project took me a around month to complete from scratch, from research, to creating flow charts & making decisions on backend functionality, testing the smart contracts with the Chai framework, frontend, and a final public testing with my friends.

This is my first time working with tokenURIs and fetching them as JSON files on the frontend, and producing 100s of JSON files with an art engine. Some problems only arised during a private testing session, such as duplicate tokenURIs when 2 users mint at the same time, due to initially setting tokenURIs on the frontend in attempt to save on gas.

It was a great experience and helped strengthen my understanding of the languages and my weaker parts that I've always avoided, such as generating and fetching TokenURIs / content and looping through to display content on the frontend. I also familiarized myself to use React hooks for a cleaner code.

Ways to improve would be 1)better code & gas optimization with smarter importing 2)more functionalities 3)implementing lazy minting for a multi-creator system 4)using the front end to create & store sell/buy orders instead of changing the state of the blockchain to save gas.
